New Topics & Concepts (Session 5)

1. Multi-page Website

So far we mostly worked on single-page examples. This project is your first multi-page website:

  • Each screen is a separate .html file.
  • The navbar links connect these pages together.
  • It feels like one website, but actually multiple files.

2. New HTML Tags We Are Using

Some tags are new in this project:

  • <nav> – used for navigation menu / navbar at the top.
  • <link> – connects an external CSS file to the HTML page.
  • <meta> – stores page information (charset, viewport, etc.).
  • <strong> – makes text bold with importance/meaning.

3. External CSS File (Shared Design)

Instead of writing CSS inside each HTML file, we are using one shared CSS file: 

<link rel="stylesheet" href="style.css" />
  • All pages share the same style.css.
  • Changing CSS in one place updates the whole site.
  • This is how most real projects are structured.

4. HTML Forms & Inputs (What We Will Build on Contact Page)

On the Contact page, we will build a real contact form together. Some new tags and attributes we will use:

  • <form> ... </form> – wraps all inputs.
    Used to group fields and handle submission.
  • <label> – text label for an input.
    Helps with usability and accessibility.
  • <input> – single-line fields like name, email, phone.
    We will use types like text and email.
  • <textarea> – multi-line text box for messages.
  • <button type="submit"> – button to submit the form.

Important Attributes We Will Use

  • placeholder="Enter your name" – short hint shown inside the input.
  • required – makes the field mandatory before submitting.
  • type="email" – tells the browser this input should be an email.
  • name="email" – the key used later when sending data to backend.

We will first see these tags and attributes here, then switch to the Contact page and build the full form live with HTML, CSS, and JavaScript.
